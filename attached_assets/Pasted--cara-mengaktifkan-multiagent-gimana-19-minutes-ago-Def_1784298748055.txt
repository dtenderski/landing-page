Defining orchestration setup
Di sistem Gustafta, multi-agent bekerja seperti ini:

Konsepnya sederhana:

Bot utama (misal ProyekBot) jadi orchestrator — ia yang menerima pertanyaan user
Lalu mendelegasikan ke sub-agen spesialis sesuai topik (misal: sub-agen khusus jadwal, sub-agen khusus EVM, sub-agen khusus kontrak)
Sub-agen menjawab, orchestrator merangkum ke user
3 cara mengaktifkan:

1. Via Admin Panel (paling mudah) Buka /admin → pilih agen → tab "Tim & Kolaborasi" atau "Sub-Agents" → tambahkan agen lain sebagai anggota tim. Platform otomatis set is_orchestrator = true dan isi agenticSubAgents.

2. Via Blueprint Builder Buka /blueprint atau dari dashboard admin → "Rancang Tim AI" → susun struktur organisasi agen (siapa orchestrator, siapa sub-agen, relasi antar agen). Ini cara paling visual dan aman karena ada validasi otomatis.

3. Langsung via database (untuk yang sudah tahu strukturnya) Set is_orchestrator = true + isi kolom agenticSubAgents dengan JSON list sub-agen beserta capability-nya.

Kapan perlu multi-agent?

Kalau satu bot makin sering ditanya hal yang sangat berbeda domain-nya — misalnya TenderBot sering ditanya soal keuangan tender DAN legalitas kontrak DAN strategi teknis sekaligus — barulah perlu dipecah ke sub-agen spesialis. Selama pertanyaan masih bisa dijawab satu sistem prompt yang fokus, standalone sudah cukup.
